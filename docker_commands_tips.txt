docker container top - список процессов в одном конейнере
docker container inspect - детали конфигурации одного контейнера
docker container stats - статистика производительности контейнеров (или одного если указать имя)
docker container run -it - запустить контейнер интерактивно
docker container exec -it - запустить дополнительно команду в существующем контейнере
docker container start -ai - запустить уже готовый контейнер. В отличие от run это не создаст новый контейнер
=========================================
docker container run -p "ports" - запускает контейнер с настроенными портами
docker container port <container> - показывает порты контейнера
---------
* Каждый контейнер подключён к частрой виртуальной сети "bridge"
* Каждая виртуальная сеть маршрутизируется через NAT firewall на host IP
* Все контейнеры на виртуальной сети могут общаться друг с другом без флага -p
* Лучшей практикой является создание виртуальной сети для каждого приложения:
 ** сеть "my_web_app" для mysql и php/apache контейнеров
 ** сеть "my_api" для mongo и nodejs контейнеров
 
docker networks ls - список виртуальных сетей
docker network inspect - посмотреть свойства сети
docker network create --driver - создать виртуальную сеть для контейнера со своими настройками
